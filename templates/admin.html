Data Management
Database Tables Overview

This database contains two types of tables:

  • Transactional Tables: Event data that changes during the read-a-thon (reading logs, donations, uploads, bonuses) - can be cleared below
  • System Tables: School structure data that stays constant (students, classes, grade rules) - managed via Database Creation tool

To update system tables for a new school year, use the Database Creation tool in the Database tab.

Create New Read-a-Thon Database
Database Creation Overview

Create a brand new read-a-thon database for a new school year. This process will:

  • Create a new database file (e.g., readathon_2027.db)
  • Initialize all required tables (Roster, Class_Info, Grade_Rules, Daily_Logs, etc.)
  • Load your school data from the three CSV files you provide
  • Register the new database in Database_Metadata for easy switching

Sample CSV files are available in the csv/ directory for reference.

Enter the year the school year ends (e.g., 2027 for 2026-2027 school year)
