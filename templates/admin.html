Data Management
Database Tables Overview

This database contains two types of tables:

  • Transactional Tables: Event data that changes during the read-a-thon (reading logs, donations, uploads, bonuses) - can be cleared below
  • System Tables: School structure data that stays constant (students, classes, grade rules) - managed via Database Creation tool

To update system tables for a new school year, use the Database Creation tool in the Database tab.

Create New Read-a-Thon Database
Database Creation Overview

Create a brand new read-a-thon database for a new school year. This process will:

  • Create a new database file (e.g., readathon_2027.db)
  • Initialize all required tables (Roster, Class_Info, Grade_Rules, Daily_Logs, etc.)
  • Load your school data from the three CSV files you provide
  • Register the new database in the central Database_Registry for easy switching

Sample CSV files are available in the csv/ directory for reference.

Enter the year the school year ends (e.g., 2027 for 2026-2027 school year)

Database Registry
About the Database Registry

The Database Registry tracks all read-a-thon databases in a separate registry database (db/readathon_registry.db ‚Üí Database_Registry table). Each contest database (like readathon_2025.db) stores its own student/reading data.

  • Active Database: The currently-in-use database. Only ONE database can be active at a time. Activating a database automatically deactivates the previous one.
  • Switching Databases: Use the dropdown in the header (top-right) OR click "Activate" below. Both methods update the active database and reload the page with the new database's data.
  • Update Stats: Recalculates summary statistics (student count, days, donations) from the contest database and updates the registry. Run this after uploading new data.
  • Register: Add an existing database file to the registry so it appears in the dropdown.

Tip: Display names below match the database selector in the header.
